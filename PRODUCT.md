# Product

## Register

product

## Users

The primary user is an operator managing automotive accessory lead collection, email outreach, mailbox review, and follow-up triage from one Windows workstation.

## Product Purpose

This tool shows current customer counts by vehicle category, starts lead collection jobs, sends outreach emails from selected mailboxes, monitors inbound replies, and separates messages that need direct human follow-up. Success means the operator can see queue health, send only available unsent leads, and react quickly to qualified customer interest.

## Brand Personality

Precise, calm, operational. The interface should feel like a dependable work console, not a marketing page or decorative demo.

## Anti-references

Avoid ornate glass effects, decorative gradients, floating card stacks, oversized hero layouts, playful consumer-app styling, and vague sales copy. The page should not look like a landing page.

## Design Principles

1. Put operational state first: counts, send readiness, mailbox status, and follow-up items must be scan-friendly.
2. Keep actions close to context: collection, send, reset, clear, and read actions should be visually distinct without shouting.
3. Favor density with breathing room: this is a repeated-use dashboard, so reduce decorative copy and preserve readable spacing.
4. Use color for state and action only: accent color should identify primary actions, unread items, success, warning, and failure.
5. Make failure visible but bounded: invalid sends and mailbox errors should be easy to find without dominating normal work.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls, preserve keyboard focus visibility, support reduced-motion preferences, and avoid relying on color alone for critical status.
